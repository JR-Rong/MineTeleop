# 需求规格

## 1. 功能需求

### 1.1 实时视频

车端需要采集多路相机画面并发送到驾驶端。

要求：

- 默认支持 4 路相机。
- 相机数量可配置，可增加或减少。
- 每路相机可独立配置：
  - 启用状态。
  - 设备路径或采集标识。
  - 采集分辨率。
  - 实时流分辨率。
  - 实时流帧率。
  - 实时流码率。
  - 关键帧间隔。
  - 编码器后端。
- 首版实时流默认使用 720p 30 fps。
- 低延迟优先，实时流不得因网络抖动无限缓存旧帧。
- 驾驶端需要显示每路视频连接状态、帧率、码率和延迟估计。

### 1.2 控制指令

驾驶端需要向车端发送控制指令。

首版控制量：

- 档位。
- 转向。
- 油门。
- 刹车。
- 急停。

要求：

- 控制频率默认 20 Hz。
- 每条控制命令包含序号和时间戳。
- 车端应丢弃过期命令。
- 车端应能检测控制心跳超时。
- 控制链路异常时车端执行安全停车。
- 第一版驾驶端使用键盘/软件控件产生控制命令。
- 后续预留方向盘、踏板、档位器等硬件输入适配。

### 1.3 车辆状态回传

车端需要向驾驶端回传基础行驶状态。

首版状态：

- 当前速度。
- 当前档位。
- 转向反馈。
- 油门反馈。
- 刹车反馈。
- 急停状态。
- 故障状态。
- 控制链路状态。
- 视频链路状态。
- 车端 CPU/GPU/内存/磁盘基础状态。

说明：

- 如果真实车辆反馈接口暂不可用，第一版可使用 Mock Telemetry。
- Mock Telemetry 必须在 UI 上清晰标识，不能伪装成真实车辆反馈。

### 1.4 本地录像

车端需要把相机原始采集画面编码保存为视频。

要求：

- 录像分辨率默认使用采集原分辨率。
- 录像流与实时流分离。
- 录像可独立配置编码器、码率、分段时长和保存目录。
- 每个视频文件需要包含可追溯元数据：
  - 车辆 ID。
  - 会话 ID。
  - 相机 ID。
  - 开始时间。
  - 结束时间。
  - 编码格式。
  - 分辨率。
  - 帧率。
  - 文件大小。
  - 上传状态。
- 磁盘空间不足时应进入保护策略，而不是挤爆系统盘。

### 1.5 批量压缩与上传

车端需要按策略把录像文件上传云端。

要求：

- 按数量、大小或时间窗口触发批量上传。
- 上传失败可重试。
- 上传过程不影响实时视频和控制链路。
- 上传限速可配置。
- 上传队列可持久化，重启后继续。
- 支持对象存储/S3 兼容接口作为默认设计。
- 如果首版还没有对象存储，可先实现本地归档或上传到测试目录的适配器。

注意：

视频文件本身已经是压缩编码，简单 zip/tar 不一定显著降低体积。真正省流量应通过录像编码 profile、分辨率、码率、保留周期和上传策略控制。

## 2. 非功能需求

### 2.1 延迟

目标：

- 设计目标：端到端视频 50-150 ms。
- 第一阶段验收：按直连、TURN UDP 中继、弱网三类场景分别记录实际数据。

说明：

- 50-150 ms 是设计目标，不保证所有 5G 网络和所有中继路径都能稳定达到。
- 必须实测编码耗时、网络 RTT、抖动、丢包和驾驶端解码显示耗时。

### 2.2 流量

目标：

- 实时流尽量低码率。
- 弱网时支持降码率、降帧率或降分辨率。
- 录像上传可限速、排队、错峰。

首版建议：

- 单路实时流 720p30 H.264，初始码率 2-4 Mbps。
- 4 路合计初始 8-16 Mbps，根据 5G 实测调整。
- 上传默认低优先级，禁止抢占实时链路。

### 2.3 可配置性

所有关键参数必须配置化：

- 相机数量和设备。
- 分辨率、帧率、码率。
- 编码器后端。
- WebRTC ICE/STUN/TURN。
- 控制频率和超时阈值。
- 安全停车参数。
- 录像分段和保留策略。
- 上传目标和限速。
- 日志级别。

### 2.4 可靠性

要求：

- 车端 Agent 开机自启动。
- 关键进程异常退出后自动拉起。
- 网络断开后自动重连。
- 云端信令断开不应导致车端失控。
- 控制超时安全停车由车端本地执行，不依赖云端。
- 录像和上传失败不能阻塞实时控制。

### 2.5 安全

要求：

- 车端设备身份使用设备证书或等价机制。
- 驾驶端登录后获取会话令牌。
- 一车一驾驶员控制权互斥。
- 所有公网通信使用 TLS 或 WebRTC DTLS/SRTP。
- 控制操作、会话开始/结束、急停、异常断开需要审计日志。

## 3. 第一版范围

包含：

- 单车单驾驶员。
- 1 路到 4 路视频推流。
- 键盘/软件控件控制。
- Mock Vehicle Adapter。
- 基础 Telemetry。
- Intel VAAPI H.264 编码验证。
- 本地录像分段保存。
- 上传队列框架。
- 云端信令和 TURN 配置。

不包含：

- 多车调度平台。
- 多驾驶员控制权移交。
- 公开道路合规。
- 自动驾驶算法。
- 完整车辆底层 CAN 协议实现。
- 完整对象存储生命周期策略。
