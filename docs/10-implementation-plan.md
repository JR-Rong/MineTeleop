# 实施计划

## 总体策略

采用分阶段实现，不一次性做完整平台。

每个阶段都必须产出可运行、可验证的软件闭环。

## Phase 0: 项目骨架和开发环境

目标：

- 建立仓库结构。
- 固定文档、配置、构建和测试入口。

建议目录：

```text
mine-teleop/
  README.md
  docs/
  vehicle-agent/
  driver-console/
  signaling-server/
  configs/
  scripts/
  tests/
  deployments/
```

验收：

- 能在本地构建空程序。
- 能加载示例配置。
- CI 或本地脚本能跑基础检查。

## Phase 1: 单路视频和控制 Mock 闭环

目标：

- 跑通车端到驾驶端的单路视频。
- 跑通驾驶端到车端的 20 Hz 控制命令。
- 车端使用 MockVehicleAdapter。

任务：

1. 实现 `signaling-server` 的登录占位和 WebSocket 信令。
2. 实现车端 TestPatternSource。
3. 实现单路 H.264 编码。
4. 建立 WebRTC 连接。
5. 驾驶端显示单路视频。
6. 驾驶端生成键盘/软件控制命令。
7. 车端接收命令并打印到 MockVehicleAdapter。
8. 实现控制超时分级减速安全停车状态。
9. 验证急停收到一次后由车端锁存，解除需要显式复位流程。

验收：

- 驾驶端看到实时视频。
- 车端日志显示 20 Hz 控制命令。
- 断开驾驶端后车端进入分级减速安全停车。
- 触发急停后即使驾驶端断连，车端仍保持 `ESTOP` 锁存。

## Phase 2: Intel VAAPI 媒体后端

目标：

- 在工控机上使用 Intel VAAPI 进行实时 H.264 编码。

任务：

1. 固定容器媒体环境。
2. 实现 VAAPI 编码器选择。
3. 增加 CPU x264 兜底。
4. 记录编码器实际生效值。
5. 跑单路真实相机或测试源。
6. 跑 4 路并发测试。

验收：

- 单路 720p30 H.264 VAAPI 成功。
- 4 路 720p30 能稳定运行。
- CPU 占用、GPU 占用、码率、fps 有记录。

## Phase 3: 多路相机和 UI

目标：

- 支持 4 路相机。
- 驾驶端显示 4 路视频网格。

任务：

1. 实现 camera config 列表。
2. 每路相机独立 pipeline。
3. 每路独立状态上报。
4. 驾驶端 1/2/4 路布局。
5. 单路断流 UI 标记。

验收：

- 任意启用 1-4 路相机。
- 单路相机故障不影响其他路。
- UI 显示每路 fps/码率/连接状态。

## Phase 4: 录像分段和上传队列

目标：

- 车端原分辨率录像。
- 分段保存。
- 上传队列持久化。

任务：

1. 实现 Record Branch。
2. 实现 Segment Writer。
3. 写 sidecar JSON 元数据。
4. 实现上传队列状态机。
5. 实现本地测试上传后端。
6. 接入 S3 兼容上传适配器。
7. 实现逐文件上传、预签名 URL 过期刷新、上传限速和重试。
8. 增加录像产生速率、磁盘容量、上传带宽和保留时长的容量规划检查。

验收：

- 每路生成分段视频和元数据。
- 重启后上传队列不丢。
- 上传失败可重试，预签名 URL 过期后可重新申请。
- 上传不会影响实时视频和控制。
- 上传追不上录像产生速率时能进入明确告警或降级策略。

## Phase 5: 安全、鉴权和审计

目标：

- 让系统具备公网基本安全边界。

任务：

1. 设备证书或设备密钥注册。
2. 驾驶端登录和会话令牌。
3. 一车一驾驶员控制权。
4. 云端审计日志。
5. 车端会话校验。
6. 急停审计。

验收：

- 未登录驾驶端不能控制车辆。
- 非当前会话命令被拒绝。
- 控制权状态可追溯。

## Phase 6: 真实车辆适配

目标：

- 从 MockVehicleAdapter 切到真实车辆接口。

前置条件：

- 明确底层接口是 CAN 还是动态库。
- 明确控制量单位、范围、频率和心跳。
- 明确底层安全停车能力。

任务：

1. 编写接口协议文档。
2. 实现适配器。
3. 离线模拟测试。
4. 台架测试。
5. 低速封闭场地测试。

验收：

- 真实车辆能执行控制。
- 控制超时能安全停车。
- 急停能可靠触发。

## 推荐优先级

最先做：

1. 单路视频。
2. 控制 Mock 闭环。
3. 超时安全停车。
4. Intel VAAPI 容器环境。

暂缓：

- 多车调度。
- 复杂权限体系。
- 多驾驶员观看。
- 真实方向盘外设。
- 完整云端录像管理后台。
